# 🤖 CAN - AI Agent Instructions

이 문서는 모바일 동기부여 플랫폼 **CAN** 개발을 지원하는 AI 어시스턴트를 위한 개발 지침 및 프로젝트 규칙 정의서입니다. 모든 코드 생성, 문서 작성, 아키텍처 제안 시 아래 정의된 규칙을 준수해야 합니다.

---

## 📌 1. 프로젝트 정체성 및 핵심 가치
* **프로젝트명**: CAN
* **플랫폼**: 모바일 전용 (Mobile Only)
* **디자인 원칙**: 
  * 극도의 단순함과 미니멀리즘 (사용자가 메시지에 온전히 몰입하도록 유도)
  * 한 손 조작 편의성 (중요 조작 인터페이스를 하단 및 엄지손가락 범위인 "Thumb-Zone" 내에 배치)
  * 다채로운 개인화 (사용자가 변경할 수 있는 테마 색상 및 다크 모드 연동)

---

## ⚙️ 2. 기술 스택 및 아키텍처 가이드라인

개발 시 아래 기술 스택 표준 및 데이터 규칙을 기반으로 코드를 설계하고 조언해야 합니다.

### 기술 스택 (가칭 - 실제 적용 스택에 맞게 수정 가능)
* **Frontend**: Flutter (또는 React Native / Swift / Kotlin)
* **Backend & Database**: Supabase (PostgreSQL) / REST API
* **State Management**: 테마 상태(Theme State) 및 전역 상태 관리를 지원하는 도구 활용 (예: Riverpod, Provider, Redux 등)

### 데이터 관리 원칙 (100,000+ 대용량 데이터 대응)
* **로컬 하드코딩 절대 금지**: 모든 명언 데이터는 클라우드 DB에서 조회합니다.
* **페이지네이션(Pagination) 필수**: 명언 리스트나 커뮤니티 피드 조회 시 반드시 오프셋(Offset) 또는 커서(Cursor) 기반 페이지네이션 코드를 구현해야 합니다.
* **감정/상황 매칭 아키텍처**: 사용자가 자연어 문장을 입력했을 때, 태그 매칭 혹은 벡터 유사도 검색(예: pgvector)을 고려한 DB 쿼리나 API 통신 코드를 우선 설계합니다.

## 📱 3. 기능별 코드 작성 지침

AI는 코드를 작성하거나 리팩토링할 때 아래 기능별 설계 기준을 반드시 따릅니다.

### ① 테마 색상 변경 (Theme Customization)
* 테마 변경 상태는 앱 전반의 위젯 트리에 실시간으로 반영되어야 합니다.
* 하드코딩된 색상 상수를 지양하고, 글로벌 테마 데이터 개체(Theme Data)를 정의하여 적용하십시오.

### ② 하단 탭 구조 (Bottom Navigation Bar)
* 5개 탭(홈, 검색/추천, 커뮤니티, 알람, 배경화면)의 상태를 효율적으로 보존해야 합니다.
* 탭 전환 시 화면이 초기화되지 않고 이전 스크롤 위치나 입력 데이터가 유지되도록 설계하십시오.
s
### ③ 커뮤니티 및 인기 차트
* 하트(좋아요) 토글 시 낙관적 업데이트(Optimistic Update) 패턴을 적용하여 네트워크 딜레이 없이 즉각 반응하는 UI를 유도하십시오.
* 인기 차트는 DB 쿼리 효율성을 높이기 위해 주기적인 캐싱이나 인덱스가 고려된 호출 방식을 권장합니다.

### ④ 모바일 친화적 알람 및 배경화면
* 알람 시간 설정 인터페이스는 모바일에 최적화된 큰 휠이나 드래그 형태를 권장합니다.
* 알람 해제 타이핑 화면은 모바일 키보드 영역(Software Keyboard)이 활성화되어도 UI 요소가 깨지거나 가려지지 않도록 스크롤 뷰 및 오버플로우 방지 처리를 하십시오.
* 배경화면 이미지 저장은 모바일 기기의 플랫폼 권한(갤러리 쓰기 권한) 처리를 반드시 포함해야 합니다.

## 🛠️ 4. AI 개발 태도 및 코드 품질 규칙

1. **간결하고 완성도 높은 코드**: 일부분만 작성하고 생략(`// TODO` 등)하는 것을 최소화하고, 구동 가능한 완전한 형태의 코드를 우선 제공하십시오.
2. **겸손하고 명확한 어조**: 기술적 제안을 할 때 과장하거나 절대적인 표현(예: "완벽한 코드", "무조건 성공하는 방식")을 피하고, 대안과 한계를 명시하십시오.
3. **가독성 높은 주석**: 복잡한 알고리즘이나 상태 관리 로직 영역에는 주석을 한글로 상세히 기술하십시오.
4. **리팩토링 우선**: 중복 코드를 방지하기 위해 공통 모바일 위젯, 유틸리티 함수, 상수 클래스 분리를 수시로 제안하십시오.
