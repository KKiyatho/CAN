# CAN 개발 MVP

## 1. 문서 목적
이 문서는 CAN의 아이디어를 실제로 개발 가능한 1차 출시 범위로 축소한 MVP 정의서입니다.

MVP의 목표는 아래 3가지를 가장 짧은 개발 사이클로 검증하는 것입니다.

1. 사용자는 매일 짧고 강한 동기부여 메시지를 쉽게 소비할 수 있는가
2. 사용자는 현재 감정이나 상황에 맞는 명언 추천을 유용하다고 느끼는가
3. 사용자는 커뮤니티, 알람, 배경화면 기능을 통해 앱을 반복적으로 다시 방문하는가

## 2. MVP 제품 원칙
- 모바일 전용으로 설계한다.
- 화면은 단순하고 빠르게 이해되어야 한다.
- 주요 조작은 하단과 엄지 범위 안에 배치한다.
- 테마 색상과 다크 모드는 앱 전역에 즉시 반영한다.
- 명언 데이터는 로컬 하드코딩 없이 Supabase에서 조회한다.
- 목록성 데이터는 모두 페이지네이션을 적용한다.
- 감정 추천은 처음부터 확장 가능한 구조로 설계하고, MVP에서는 태그 기반 매칭을 우선 적용한다.

## 3. 권장 기술 스택
MVP 기준 권장 조합은 아래와 같습니다.

- 앱: Flutter
- 상태 관리: Riverpod
- 백엔드/DB: Supabase PostgreSQL
- 인증: Supabase Anonymous Auth 또는 간단한 이메일 로그인
- 파일 저장: Supabase Storage
- 알람/로컬 알림: Flutter Local Notifications
- 배경화면 이미지 저장: 모바일 저장 권한 플러그인

선정 이유는 모바일 UI 구현 속도, 테마 전역 관리, Supabase 연동 편의성, 추후 iOS/Android 동시 출시 대응 때문입니다.

## 4. MVP 핵심 사용자 시나리오
### 시나리오 1. 오늘의 명언 확인
사용자는 앱을 열자마자 오늘의 명언 카드 1개를 보고, 북마크 또는 공유를 누를 수 있어야 합니다.

### 시나리오 2. 감정 기반 추천 받기
사용자는 검색 탭에서 현재 감정이나 상황을 입력하고, 그 입력과 매칭되는 명언 목록을 즉시 받아야 합니다.

### 시나리오 3. 커뮤니티에서 공감하기
사용자는 커뮤니티 글 목록을 보고 하트를 누를 수 있어야 하며, 반응은 낙관적 업데이트로 즉시 반영되어야 합니다.

### 시나리오 4. 알람 설정 후 해제하기
사용자는 휠 기반 UI로 알람을 설정하고, 알람이 울리면 지정된 문구를 입력해 해제할 수 있어야 합니다.

### 시나리오 5. 명언 배경화면 저장하기
사용자는 명언과 배경을 선택한 뒤 이미지를 생성해 기기에 저장할 수 있어야 합니다.

## 5. MVP 범위
### 5.1 반드시 포함할 기능
#### 1) 공통
- 온보딩 1회: 기본 테마 색상 선택
- 라이트/다크 모드 지원
- 하단 5탭 구조
- 탭 전환 시 상태 유지
- 로딩, 빈 상태, 오류 상태 UI 제공

#### 2) 홈 탭
- 오늘의 명언 카드 1개 노출
- 새로고침 또는 다른 명언 보기
- 북마크 저장
- 공유 버튼

#### 3) 검색/추천 탭
- 키워드 검색
- 감정/상황 입력창
- 태그 기반 추천 결과 목록
- 최근 검색어 5개 저장
- 결과 목록 페이지네이션

#### 4) 커뮤니티 탭
- 게시글 목록 조회
- 게시글 작성
- 하트 토글 낙관적 업데이트
- 월간 인기 차트 상위 10개 노출
- 피드 페이지네이션

#### 5) 알람 탭
- 알람 1개 생성, 수정, 삭제
- 반복 요일 선택
- 명언 타이핑 해제 화면
- 로컬 알림 기반 동작

#### 6) 배경화면 탭
- 명언 선택
- 배경 이미지 또는 배경색 선택
- 폰트 크기 3단계 조절
- 미리보기
- 기기 저장 권한 요청 및 저장

### 5.2 MVP에서 제외할 기능
- 소셜 로그인
- 실시간 채팅
- 팔로우 시스템
- 복수 알람 고급 스케줄링
- AI 생성 명언
- 벡터 검색 기반 정교한 추천
- 홈 화면 위젯
- 관리자 CMS

## 6. MVP 화면 정의
### 6.1 홈
- 상단: 앱 로고 또는 오늘의 메시지 제목
- 중앙: 큰 명언 카드
- 하단: 북마크, 공유, 다음 명언 버튼

### 6.2 검색/추천
- 상단: 검색창 + 감정 입력창
- 중단: 추천 태그 칩 또는 추천 이유
- 하단: 결과 리스트

### 6.3 커뮤니티
- 상단: 월간 인기 차트 진입 영역
- 중단: 글 작성 버튼
- 하단: 피드 리스트 + 하트 버튼

### 6.4 알람
- 상단: 활성화 스위치
- 중단: 휠 시간 선택기
- 하단: 반복 요일, 해제 문구 미리보기

### 6.5 배경화면
- 상단: 템플릿 선택
- 중단: 미리보기 캔버스
- 하단: 폰트 크기, 배경, 저장 버튼

## 8. 추천 로직 MVP 정의
MVP에서는 복잡한 AI 추천보다 빠르게 검증 가능한 규칙 기반 추천을 사용합니다.

1. 사용자가 입력한 문장에서 핵심 키워드를 추출한다.
2. 키워드를 사전에 정의한 감정 태그와 매칭한다.
3. quotes와 quote_tags를 기준으로 관련 명언을 조회한다.
4. 결과가 없으면 인기 명언 또는 오늘의 추천 명언으로 폴백한다.

예시 태그:
- 불안
- 지침
- 도전
- 면접
- 관계
- 자기계발

이 구조는 이후 pgvector 기반 유사도 검색으로 교체 가능해야 합니다.

## 9. API 또는 데이터 접근 규칙
### 9.1 목록 조회
- 홈 제외 모든 목록은 limit/offset 또는 cursor 기반 페이지네이션 필수
- 기본 페이지 크기: 20

### 9.2 좋아요 처리
- 클라이언트는 먼저 UI를 갱신한다.
- 실패 시 이전 상태로 롤백한다.

### 9.3 인기 차트
- 실시간 집계 대신 일 단위 또는 시간 단위 캐시 허용
- 월간 집계용 뷰 또는 배치 테이블 사용 권장

### 9.4 명언 데이터
- 앱 번들 내부 상수 배열로 저장하지 않는다.
- 초기 시드 데이터는 DB 마이그레이션 또는 CSV import로 적재한다.

## 11. 1차 릴리스 기준 수용 조건
아래 조건을 모두 만족하면 MVP 출시 가능 상태로 판단합니다.

1. 앱이 iOS와 Android 실기기에서 모두 실행된다.
2. 5개 탭이 동작하고 탭 상태가 유지된다.
3. 오늘의 명언, 검색 결과, 커뮤니티 목록이 모두 Supabase 데이터로 로드된다.
4. 커뮤니티 좋아요가 즉시 반응하고 실패 시 복구된다.
5. 검색/추천 결과가 페이지네이션된다.
6. 테마 색상 변경과 다크 모드가 앱 전역에 반영된다.
7. 알람 설정과 해제 타이핑 흐름이 모바일에서 깨지지 않는다.
8. 배경화면 저장 권한 요청과 이미지 저장이 실제 기기에서 동작한다.

## 13. 구현 메모
- 디자인은 화려함보다 집중감을 우선한다.
- 하단 조작 영역은 큰 터치 타깃을 유지한다.
- 색상은 상수 하드코딩 대신 ThemeExtension 또는 동등한 전역 테마 객체로 관리한다.
- 커뮤니티와 검색 결과는 무한 스크롤 또는 더보기 방식 중 하나로 반드시 페이지네이션한다.
- 복잡한 추천 엔진을 먼저 만들기보다, 태그 품질과 기본 추천 경험을 먼저 검증한다.

## 14. 한 줄 결론
CAN의 MVP는 오늘의 명언 소비, 감정 기반 추천, 커뮤니티 공감, 알람 재방문, 배경화면 저장까지를 하나의 모바일 흐름으로 묶되, 추천 로직과 운영 기능은 단순하게 시작하는 것이 핵심입니다.
